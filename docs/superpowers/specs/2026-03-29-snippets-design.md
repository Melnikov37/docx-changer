# Справочники DOCX-фрагментов

## Назначение

Пользователи создают справочники с DOCX-фрагментами (таблицы, абзацы с форматированием), организованные по категориям. При генерации документа фрагменты вставляются в шаблон по специальным меткам.

Пример: категория "Туроператоры" содержит записи с таблицами сведений о реестровых номерах, финансовом обеспечении и т.д. При генерации договора пользователь выбирает нужного туроператора, и его данные вставляются в отмеченное место шаблона.

## Модель данных

### Таблица `snippet_categories`

```sql
CREATE TABLE snippet_categories (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### Таблица `snippets`

```sql
CREATE TABLE snippets (
    id SERIAL PRIMARY KEY,
    category_id INTEGER NOT NULL REFERENCES snippet_categories(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    description TEXT,
    s3_key TEXT NOT NULL UNIQUE,
    original_filename TEXT NOT NULL,
    file_size INTEGER,
    user_id INTEGER NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

Связь: категория 1 → N фрагментов. Каскадное удаление записей при удалении категории. Пользователь видит только свои данные (фильтрация по `user_id`).

## Механизм вставки

### Метки в шаблоне

В DOCX-шаблоне используется формат:

```
{{SNIPPET:имя_метки}}
```

Примеры: `{{SNIPPET:туроператор}}`, `{{SNIPPET:страховая}}`, `{{SNIPPET:финобеспечение}}`

### Парсинг

При загрузке шаблона система распознаёт метки `SNIPPET:*` отдельно от обычных Jinja2-переменных. Регулярное выражение:

```python
snippet_pattern = r'\{\{\s*SNIPPET\s*:\s*([a-zA-Zа-яА-ЯёЁ0-9_]+)\s*\}\}'
```

Результат парсинга возвращает и обычные переменные, и список SNIPPET-меток.

### Генерация документа

1. Обычные переменные заполняются через `docxtpl` (как сейчас)
2. Для каждой SNIPPET-метки:
   - Находим параграф с меткой в документе
   - Загружаем выбранный DOCX-фрагмент из S3
   - Вставляем все элементы фрагмента (параграфы, таблицы) на место метки через `python-docx`
   - Удаляем параграф с меткой

Если для метки выбрано "Не вставлять" — параграф с меткой просто удаляется.

## API-эндпоинты

### Категории

- `GET /snippets/categories` — список категорий пользователя
- `POST /snippets/categories` — создание категории (name, description)
- `PUT /snippets/categories/<id>` — обновление категории
- `DELETE /snippets/categories/<id>` — удаление категории (каскадно удаляет записи и файлы из S3)

### Записи-фрагменты

- `GET /snippets/categories/<cat_id>/items` — список записей в категории
- `GET /snippets/items` — все фрагменты пользователя (для dropdown при генерации), сгруппированные по категориям
- `POST /snippets/items` — создание записи (name, description, category_id, file)
- `GET /snippets/items/<id>/preview` — HTML-превью содержимого фрагмента
- `GET /snippets/items/<id>/download` — скачивание оригинального DOCX
- `PUT /snippets/items/<id>` — обновление метаданных или замена файла
- `DELETE /snippets/items/<id>` — удаление записи и файла из S3

### Создание из формы

- `POST /snippets/items/from-form` — создание записи через веб-форму: принимает набор полей (ключ-значение), генерирует DOCX-таблицу и сохраняет

## UI

### Страница `/snippets`

Новая страница, доступна из навигации (ссылка в navbar).

**Левая/верхняя часть — категории:**
- Список категорий как карточки (название, количество записей, описание)
- Кнопка "Создать категорию" → модальное окно с полями: название, описание
- Кнопка удаления на каждой карточке (с подтверждением)

**При выборе категории — список записей:**
- Таблица или карточки: название, описание, размер файла, дата
- Кнопки на каждой записи: Просмотр, Скачать, Заменить файл, Удалить

**Модальное окно создания записи:**
- Название, описание
- Выбор категории (dropdown)
- Два таба:
  - "Загрузить файл" — drag-and-drop зона для .docx
  - "Заполнить форму" — динамические поля ключ-значение, из которых система генерирует DOCX-таблицу

**Модальное окно просмотра:**
- HTML-рендер содержимого DOCX (таблицы, текст)
- Кнопка "Скачать оригинал"
- Кнопка "Заменить файл" (загрузить новый .docx)

### Изменения на главной странице (генерация)

При загрузке шаблона с метками `{{SNIPPET:*}}`:
- В динамической форме для каждой SNIPPET-метки появляется секция с dropdown
- Dropdown сгруппирован по категориям (optgroup)
- Первая опция — "Не вставлять"
- Секции SNIPPET визуально выделены (другой фон/рамка)

## Просмотр DOCX как HTML

Для preview используем `python-docx`:
- Извлекаем параграфы и таблицы
- Конвертируем в HTML с базовыми стилями (шрифт, жирный, курсив, таблицы с рамками)
- Точная передача форматирования не гарантируется — это превью, не рендер

## Создание из формы

Когда пользователь выбирает "Заполнить форму":
- Динамические поля ключ-значение (кнопка "Добавить поле")
- Система генерирует DOCX с таблицей из двух колонок: ключ | значение
- Сохраняет в S3 как обычный фрагмент

## Обработка ошибок

- Загрузка не-.docx файла → ошибка 400
- Файл > 10 MB → ошибка 400
- Удаление категории с записями → каскадное удаление с подтверждением
- Попытка использовать чужой фрагмент → 404
- S3 недоступен → ошибка 500 с понятным сообщением
