## Contributing to Documentation

When adding new documentation:

1. Place files in appropriate subdirectories:
   - `deployment/` - Deployment and infrastructure guides
   - `guides/` - User guides and tutorials (prefer Russian)
   - Root `docs/` - Developer references and project docs

2. Update this README.md with links to new documents

3. Keep CLAUDE.md as the single source of truth for developers

4. Follow markdown best practices:
   - Use clear headings
   - Include code examples
   - Add cross-references
   - Keep language consistent per file (English or Russian)

## Documentation Standards

- **English**: Technical documentation, API references, architecture
- **Russian**: User guides, tutorials, deployment instructions for Russian users
- **Code Comments**: Always in Russian (project standard)
- **Commit Messages**: English preferred
